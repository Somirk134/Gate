# GitHub Homepage Optimization

Use this checklist when configuring the GitHub repository page.

## Repository Description

```text
Self-hosted tunnel infrastructure for exposing private TCP and HTTP services through your own server.
```

## Website

Use the documentation or website URL when available:

```text
https://lancemorii-git.github.io/gate
```

If the website is not published yet, leave the field empty rather than linking to a placeholder.

## About Panel

Enable:

- Releases.
- Packages when Docker images are published.
- Discussions.
- Security policy.

Keep the About text aligned with the repository description.

## Social Preview

Use:

```text
assets/branding/social-card.svg
```

Before launch, export a PNG version if GitHub does not render the SVG preview as expected.

## GitHub Banner

Use:

```text
assets/branding/banner.svg
```

Recommended placements:

- README top or secondary hero area.
- Website landing page.
- Release announcement.

## Release

Use `.github/RELEASE_TEMPLATE.md` for every release. Each release should include:

- Highlights.
- Added.
- Improved.
- Fixed.
- Breaking Changes.
- Known Issues.
- Upgrade Guide.

## First Impression Checklist

- README first viewport shows logo, one-liner, screenshot, badges, and primary links.
- Installation can be found in under 10 seconds.
- Docker command is visible.
- FAQ answers FRP, HTTPS, self-hosting, free, upgrade, backup, and migration.
- Examples and Cookbook are linked from README.
- License is visible.
- Contribution path is visible.
