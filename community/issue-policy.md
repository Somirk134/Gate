# Issue Policy

Use issues for actionable work.

## Good Issues Include

- Clear title.
- Version, platform, and environment.
- Steps to reproduce.
- Expected and actual behavior.
- Logs or screenshots when useful.

## Not For Issues

- Security vulnerabilities. Use `SECURITY.md`.
- General questions. Use discussions.
- Large roadmap proposals without prior discussion.
- Support requests missing reproduction details.

## Labels

| Label | Meaning |
| --- | --- |
| `bug` | Confirmed or likely defect |
| `feature` | Feature request |
| `docs` | Documentation change |
| `ci` | Automation or workflow change |
| `security` | Public security hardening task, not vulnerability details |
| `good first issue` | Maintainer-approved beginner task |
