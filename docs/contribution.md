# Contribution

Thank you for considering a contribution to Gate.

## Good First Contributions

- Improve documentation clarity.
- Add examples.
- Add practical examples and recipes.
- Reproduce issues.
- Add tests around existing behavior.
- Improve release and packaging scripts.
- Fix typos, links, and screenshots.

## Before You Start

1. Read [CONTRIBUTING.md](../CONTRIBUTING.md).
2. Search existing issues and discussions.
3. Open an issue for non-trivial changes.
4. Keep the pull request focused.

## Documentation Contributions

Documentation should:

- Use lowercase kebab-case filenames.
- Include copyable commands.
- Avoid real secrets.
- State alpha limitations clearly.
- Link to related guides.
- Update README links if the first-run path changes.

## Code Contributions

Code changes should:

- Keep runtime, protocol, and UI concerns separate.
- Follow existing crate boundaries.
- Include tests for behavior changes.
- Avoid broad refactors inside feature or bugfix PRs.
- Update docs for public behavior changes.

## Review Expectations

Maintainers prioritize:

- Correctness.
- Safety.
- Backward compatibility.
- Clear UX.
- Tests.
- Documentation.

## Community

Follow [CODE_OF_CONDUCT.md](../CODE_OF_CONDUCT.md) in issues, discussions, pull requests, and reviews.
