# Plugin Guide

Plugin support is reserved.

## Goals

- Define stable extension points.
- Protect runtime security and performance.
- Version plugin contracts explicitly.
- Provide examples before accepting third-party plugins.

## Planned Areas

| Area | Description |
| --- | --- |
| Runtime hooks | Observe tunnel lifecycle events |
| Monitoring exporters | Export metrics to external systems |
| UI panels | Add desktop dashboard extensions |
| Auth providers | Integrate external identity systems |

## Compatibility Policy

Compatibility rules will be published before v1.5. Until then, internal APIs are not stable plugin
contracts.
