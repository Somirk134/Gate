# Design

This directory contains design system notes, diagrams, and future product design references.

## Files

| File | Purpose |
| --- | --- |
| `design-system.md` | Visual and interaction guidelines |
| `architecture-map.md` | System diagram notes |

## Principles

- Operational interfaces should be dense, calm, and easy to scan.
- Use consistent status colors and avoid ambiguous states.
- Prefer explicit labels for risky actions.
- Keep documentation diagrams readable in dark mode.
